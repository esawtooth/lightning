# Vextir CLI - Complete Implementation

## Overview

The Vextir CLI is a comprehensive command-line interface for operating the Vextir AI Operating System. It provides full access to all system functionality through an intuitive, well-structured command interface.

## Architecture

### Core Components

1. **Main CLI (`main.py`)** - Entry point with Click-based command structure
2. **Configuration Management (`config.py`)** - Handles settings and authentication
3. **API Client (`client.py`)** - Communicates with Vextir OS backend
4. **Utilities (`utils.py`)** - Helper functions for formatting and data processing

### Command Structure

```
vextir
├── event          # Event management
│   ├── emit       # Emit events to the event bus
│   ├── list       # List recent events
│   └── stream     # Stream events in real-time
├── driver         # Driver management
│   ├── list       # List registered drivers
│   ├── start      # Start a driver
│   ├── stop       # Stop a driver
│   └── status     # Get driver status
├── model          # AI model management
│   ├── list       # List available models
│   └── info       # Get model details
├── tool           # Tool management
│   └── list       # List available tools
├── context        # Context hub operations
│   ├── read       # Read from context
│   ├── write      # Write to context
│   └── query      # Query context with SQL
├── instruction    # Instruction management
│   ├── list       # List instructions
│   └── execute    # Execute an instruction
├── system         # System management
│   ├── status     # Get system status
│   └── metrics    # Get system metrics
└── config         # Configuration management
    ├── get        # Get configuration values
    ├── set        # Set configuration values
    ├── delete     # Delete configuration keys
    └── reset      # Reset to defaults
```

## Features

### 1. Event Management
- **Emit Events**: Send events to the Vextir OS event bus with metadata
- **List Events**: View recent events with filtering options
- **Stream Events**: Real-time event monitoring with filters
- **Multiple Formats**: Table, JSON, and tree view outputs

### 2. Driver Control
- **Driver Discovery**: List all registered drivers with status
- **Lifecycle Management**: Start and stop drivers with configuration
- **Status Monitoring**: Detailed driver health and performance metrics
- **Filtering**: Filter by type, status, and capabilities

### 3. Model Operations
- **Model Catalog**: Browse available AI models
- **Model Details**: Get comprehensive model information
- **Provider Filtering**: Filter by AI provider (OpenAI, Anthropic, etc.)
- **Capability Search**: Find models by specific capabilities

### 4. Context Hub Access
- **Read Operations**: Retrieve content from context paths
- **Write Operations**: Store content with metadata
- **SQL Queries**: Powerful querying with SQL syntax
- **Path Management**: Hierarchical content organization

### 5. System Administration
- **Health Monitoring**: Real-time system status
- **Performance Metrics**: Detailed system metrics
- **Component Status**: Individual component health checks
- **Resource Usage**: Memory, CPU, and storage monitoring

### 6. Configuration Management
- **Flexible Storage**: JSON-based configuration
- **Environment Support**: Multiple configuration profiles
- **Authentication**: Azure CLI and token-based auth
- **Default Settings**: Sensible defaults with easy customization

## Authentication

### Azure CLI (Recommended)
```bash
az login
vextir config set auth.method azure_cli
```

### Token-Based
```bash
vextir config set auth.method token
vextir config set auth.token your_bearer_token
```

## Configuration

### Default Configuration Location
- **macOS/Linux**: `~/.vextir/config.json`
- **Windows**: `%USERPROFILE%\.vextir\config.json`

### Configuration Schema
```json
{
  "endpoint": "https://your-vextir-deployment.azurewebsites.net",
  "auth": {
    "method": "azure_cli",
    "tenant_id": null,
    "client_id": null,
    "token": null
  },
  "output": {
    "format": "table",
    "colors": true,
    "verbose": false
  },
  "event_streaming": {
    "buffer_size": 100,
    "timeout": 30
  },
  "context_hub": {
    "default_path": "/",
    "max_query_results": 1000
  }
}
```

## Usage Examples

### Event Operations
```bash
# Emit a user action event
vextir event emit user.login --metadata '{"user_id": "123", "source": "web"}'

# List recent events
vextir event list --limit 50 --type user.*

# Stream events in real-time
vextir event stream --type system.error
```

### Driver Management
```bash
# List all drivers
vextir driver list

# Start email agent
vextir driver start email_agent --config '{"enabled": true, "check_interval": 300}'

# Check driver status
vextir driver status email_agent

# Stop driver
vextir driver stop email_agent
```

### Context Operations
```bash
# Read project context
vextir context read /Projects/Alpha/status

# Write task list
vextir context write /Tasks/Daily "- Review PRs\n- Update docs" --metadata '{"priority": "high"}'

# Query for project files
vextir context query "SELECT * FROM context WHERE path LIKE '/Projects/%' AND metadata->>'type' = 'file'"
```

### System Monitoring
```bash
# Check overall system health
vextir system status

# Get detailed metrics
vextir system metrics

# Monitor continuously
watch -n 5 vextir system status
```
